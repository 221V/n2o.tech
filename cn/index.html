抽象

N2O ECO定义了完整的堆栈规范以及函数式编程语言中的参考实现。 它提供WebSocket和MQTT连接以及简约的企业协议栈：Web框架，业务流程管理，可扩展的数据存储。 作为示例，本文提出了三种N2O实现功能语言：SML，Haskell和Erlang。

对作者的建议

每种语言，N2O都在实现，应该最自然地嵌入N2O和紧凑。 如果需要某种语言的基础库之间的某个层，则可以提供它， 但如果可能，应该将其减少到零。 在某些情况下，基础 库的某些部分可以用更好的替换替换。 N2O应该提供通常 以不同的客户端语言集实现的客户端伴侣库：JavaScript， Swift和Kotlin。 如果你做的一切正确，N2O在任何语言 中都不应超过500 LOC。 正确的方法是使用相同的代码库实现 WebSocket和静态HTTP服务器。

O1 ECO

O1 ECO是标准ML N2O实施。 标准ML是一种经过认证的语言， 具有正式定义的操作语义。 这是一个简单的CML循环与INetSock.TCP 衍生的接受器。 足够简单，可以处理TCP服务器内的N2O协议。 O1 ECO提供基本的WebSocket连接和基本的应用服务器（N2O）。

  • o1/n2o — N2O 0.11 ML

O3 ECO

O3 ECO为Haskell编程语言引入了N2O协议。 它包括在Haskell中完全实现Nitrogen Web Framework（NITRO）以及由WebSocket和静态HTTP服务器（N2O）驱动的HTML示例（SAMPLE）。

  • o3/n2o — N2O 0.11 HS
  • o3/nitro — NITRO 0.11
  • o3/sample — SAMPLE 0.11

SYNRC ECO

SYNRC ECO定义了生态系统，银行，消息传递和平台开发的全栈规范和实现。 它指定了扩展应用程序，协议，进程，服务和存储的方法。 准备好高速， 低延迟的MQTT，TCP，QUIC，UDP和WebSocket应用程序，此版本的N2O ECO提供： 业务流程管理（BPE），氮网络框架（NITRO），WebSocket和MQTT连接（N2O）， REST 框架，快速简单的包管理器（MAD），用于API开发的类型驱动的BERT解析器生成器， 可伸缩数据存储（KVX）等等。

  • synrc/n2o — N2O 5.11.1 MQTT WS
  • synrc/kvx — KVX 5.11
  • synrc/mad — MAD 4.11
  • synrc/bpe — BPE 3.11
  • synrc/nitro — NITRO 3.10
  • synrc/rest — REST 5.10
  • synrc/active — ACTIVE 4.11
  • synrc/fs — FS 4.10
  • synrc/sh — SH 2.11
  • synrc/bert — BERT 1.11.1
  • synrc/review — REVIEW 1.11 TT
  • synrc/sample — REVIEW 1.11 WS
